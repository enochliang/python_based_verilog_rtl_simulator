# RTL Fault Simulator

[cite_start]The **RTL Fault Simulator** is a specialized framework designed for fault simulation and reliability analysis of Register Transfer Level (RTL) circuit designs[cite: 1, 4]. [cite_start]By performing static analysis on XML Abstract Syntax Trees (AST) generated by Verilator and combining it with dynamic logic value simulation, the tool calculates fault propagation across registers under real-world workloads[cite: 4]. [cite_start]The final output includes **ACE (Architecturally Correct Execution)** analysis reports[cite: 4].

## System Architecture & Data Flow

[cite_start]The framework operates through a **three-phase pipeline** to transform raw Verilog code into actionable reliability data[cite: 14, 15]:


### Phase 1: AST Preprocessing
* [cite_start]**Input:** Verilator XML AST[cite: 17].
* [cite_start]**Operations:** Removes non-essential nodes (integer, initial, parameter), merges aliased signals, marks signal types (register/wire), and performs **topological sorting** of sub-circuits[cite: 17].
* [cite_start]**Output:** A "clean" scheduled XML AST and sub-circuit order list[cite: 17].

### Phase 2: Python AST Construction
* [cite_start]**Input:** Scheduled AST from Phase 1[cite: 20].
* [cite_start]**Operations:** Converts XML nodes into custom Python objects (`Verilog_AST_*`), flattens array signals, and loads cycle-by-cycle logic values[cite: 20].
* [cite_start]**Output:** A `Verilog_AST` object tree ready for simulation[cite: 20].

### Phase 3: Simulation & Analysis
* [cite_start]**Input:** Object tree, logic values, and RW Table[cite: 22].
* [cite_start]**Operations:** Traverses sub-circuits based on the schedule, calculates node outputs using a 3-state logic library, propagates Fault Lists, and generates ACE/un-ACE statistics[cite: 22].
* [cite_start]**Output:** ACE results (JSON), RW Table (CSV), and control reduction reports[cite: 22].

## Core Technical Concepts

### Three-State Signal Model
[cite_start]Each `Verilog_AST_Circuit_Node` maintains three distinct value fields to track state across clock cycles[cite: 44, 45]:
* [cite_start]**`value`**: Current cycle combinational logic value[cite: 46].
* [cite_start]**`cur_value`**: Latched register value at the start of the cycle[cite: 46].
* [cite_start]**`next_value`**: Updated register value at the end of the cycle, which becomes the next cycle's `cur_value`[cite: 46].

### Propagation Probability Calculation
[cite_start]The tool calculates propagation probability using the following logic[cite: 69]:
* [cite_start]**Mask Probability:** $$mask\_probs = [1 - prob \text{ for } (dst\_reg, prob) \in (write\_events \cup ctrl\_events)]$$ [cite: 70]
* [cite_start]**Total Mask Probability:** $$tot\_m\_probs = \prod mask\_probs$$ [cite: 71, 73]
* [cite_start]**Final Propagation Probability:** $$tot\_p\_probs = 1 - tot\_m\_probs$$ [cite: 72, 74]

## Quick Start Guide

### 1. Requirements
* [cite_start]Python 3.9+ [cite: 114]
* [cite_start]`lxml`, `pandas` [cite: 115, 116]
* [cite_start]Verilator 4.x/5.x [cite: 117]

### 2. Generate AST
[cite_start]Use Verilator to produce the XML representation of your design[cite: 120]:
```bash
[cite_start]verilator -xml-output --bbox-sys -Wall --top-module <TopModule> <Design>.v [cite: 121, 122]
```

### 3. Run Preprocessing
```bash
[cite_start]python ast_schedule.py -f obj_dir/V<TopModule>.xml [cite: 125]
```

### 4. Execute Fault Simulation
```python
from ast_sim_prepare import SimulatorPrepare
# Initialize and load values
[cite_start]sim = SimulatorPrepare(ast, logic_value_file_dir='./logic_values/') [cite: 138, 140]
[cite_start]sim.preprocess() [cite: 143]
[cite_start]sim.load_logic_value(cycle=5) [cite: 147]
[cite_start]sim.init_fault_list() [cite: 148]
```

## Known Issues & Limitations
* [cite_start]**`rtl_functions.py`**: `val_neq()` contains dead code [cite: 160][cite_start], and `val_sel()` has a `NameError` (undefined `width` variable) when `start_bit == 0`[cite: 161].
* [cite_start]**`ast_node_define.py`**: The `_children` attribute is currently a class variable; it should be initialized in `__init__` to avoid shared state between instances[cite: 163].
* [cite_start]**`rw_split.py`**: Currently an empty skeleton module[cite: 158].
* [cite_start]**Naming**: `dataloader.py` uses regex to strip `genblk` prefixes, which may require updates for complex nested generate blocks[cite: 165].
